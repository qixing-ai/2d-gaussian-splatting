# 2D高斯散射AbsGS增强

## 一、问题背景

在2D高斯散射（2DGS）训练过程中，存在一个被称为"梯度碰撞"的问题。当一个大的高斯覆盖多个像素时，不同像素可能产生方向相反的梯度，这些梯度在累积时会相互抵消，导致总梯度被低估，特别是在细节丰富的区域（如对象边缘、复杂纹理）。

梯度碰撞的后果：
- **稠密化不足**：应该分裂的高斯点因梯度被低估而不触发分裂
- **细节模糊**：边缘和纹理区域的重建质量下降
- **过度平滑**：场景整体显得过于平滑，缺乏细节

## 二、AbsGS解决方案

AbsGS（Absolute Gradient Gaussian Splatting）的核心思想是：**在梯度统计过程中使用梯度的绝对值累加，而非梯度范数**。

### 原理对比

#### 传统方法（梯度范数）
```
梯度1 = (1.0, 0.0)
梯度2 = (-1.0, 0.0)
累加后范数 = ||梯度1 + 梯度2|| = ||(0.0, 0.0)|| = 0.0
```
结果：梯度被完全抵消，无法触发分裂。

#### AbsGS方法（绝对值累加）
```
梯度1 = (1.0, 0.0)
梯度2 = (-1.0, 0.0)
绝对值和 = |1.0| + |-1.0| = 2.0
```
结果：保留了梯度信息，可正确触发分裂。

## 三、实现代码

只需修改`scene/gaussian_model.py`中的`add_densification_stats`函数：

```python
def add_densification_stats(self, viewspace_point_tensor, update_filter):
    # 原始2DGS方法 - 使用梯度范数
    # self.xyz_gradient_accum[update_filter] += torch.norm(viewspace_point_tensor.grad[update_filter], dim=-1, keepdim=True)
    
    # AbsGS改进方法 - 计算绝对值和而非范数
    grad = viewspace_point_tensor.grad[update_filter]
    grad_abs = torch.abs(grad)
    self.xyz_gradient_accum[update_filter] += torch.sum(grad_abs, dim=-1, keepdim=True)
    
    self.denom[update_filter] += 1
```

## 七、技术原理详解

### 1. 梯度碰撞问题

在2DGS中，稠密化（densification）过程依赖于累积的梯度信息。当高斯点覆盖多个像素，且这些像素具有不同的目标值（例如在对象边缘）时：

- 一些像素可能要求高斯向一个方向移动以减少误差
- 另一些像素可能要求高斯向相反方向移动
- 累积这些梯度时，方向相反的分量会相互抵消

### 2. AbsGS的数学表达

假设一个高斯影响了多个像素，对应损失函数为：
$$L = \sum_j L_j$$

传统方法计算梯度范数：
$$g_i = \left\| \sum_j \frac{\partial L_j}{\partial \mu_i} \right\|$$

AbsGS方法计算绝对值和：
$$\hat{g}_i = \sum_j \left| \frac{\partial L_j}{\partial \mu_i} \right|$$

其中$\mu_i$表示第$i$个高斯的参数。

### 3. 与稠密化流程的结合

1. 训练过程中，累积每个高斯点的绝对梯度和
2. 定期使用累积的梯度信息来决定哪些高斯需要分裂
3. 梯度大且尺寸大的高斯进行分裂，梯度大但尺寸小的高斯进行复制
4. 这种更精确的稠密化决策导致更好的细节重建

## 八、其他注意事项

1. **计算效率**：
   - AbsGS的修改非常轻量，不会增加显著的计算负担
   - 理论上，计算绝对值和的算力需求与计算范数相近

2. **最佳应用场景**：
   - 细节丰富的场景：如复杂纹理、精细结构
   - 高对比度边缘：如前景/背景分界线
   - 复杂几何：如植物、头发等细小结构

3. **与其他增强的兼容性**：
   - AbsGS完全兼容其他2DGS增强，如边缘感知法向损失和多尺度SSIM
   - 可以与这些增强同时使用，获得综合效果
